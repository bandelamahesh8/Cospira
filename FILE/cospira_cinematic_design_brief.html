Module 01 · Motion Architecture
The Complete Motion System
From first pixel render to final scroll position — every state is choreographed
▶
Stage 1 — Preloader Sequence
▾
0ms — 200ms
Black frame hold
Full black. No flash. Browser paint completes silently. document.body is visibility: hidden until JS confirms all critical resources loaded.
200ms — 800ms
Wordmark construction
"COSPIRA" renders character by character using clipPath reveal. Each letter slides up from 120% Y with 40ms stagger. Spring: stiffness 280, damping 22.
800ms — 1600ms
Progress arc + counter
SVG arc traces 0°→360° in 800ms. Numeric counter increments 0→100 using GSAP.to({val}) with snap: 1. Counter uses tabular-nums font variant for stability.
1600ms — 2200ms
Portal open transition
Preloader div scales to scaleX(0) using a staggered wipe (left panel then right panel) over 600ms. ease: "expo.inOut". Page content already positioned behind it, revealed by the wipe.
2200ms+
Hero entrance fires
GSAP timeline begins. ScrollTrigger.refresh() called after a single rAF to let layout settle before scroll positions are calculated.
↓
Stage 2 — Scroll Motion Grammar
▾
A
Scrub animations (position-locked)
All section transitions use scrub: 1.2 — the animation progress is a direct function of scroll position with 1.2s lag. User controls the playhead. No auto-play surprises.
B
Section pin sequences
Game Arcade, Watch Together, and How It Works sections pin with pin: true, pinSpacing: true. Content animates while viewport is locked. Gives the illusion of 3D depth on a flat scroll.
C
Staggered grid reveals
Every card grid uses gsap.from(".card", {stagger: 0.08, y: 40, opacity: 0}) triggered by IntersectionObserver at threshold 0.1. First card at 0ms. Last card at N×80ms.
D
Horizontal scroll strip
Game arcade section uses ScrollTrigger horizontal pinning. As user scrolls down, the game cards slide left. Velocity tied to vertical scroll delta. Exit: vertical scroll resumes.
E
Text character splitting
All H1/H2 headings are processed through a character splitter on DOM ready. Each span.char has data-index. Reveal uses clipPath: inset(0 100% 0 0) → inset(0 0% 0 0) with 18ms inter-character stagger.
≋

Module 02 · 3D & Spatial Design
Three.js Hero & Parallax Architecture
The spatial layer that makes the screen feel like a window, not a wall
◻
Three.js Hero Asset — The Neural Mesh
▾
Concept
A dynamic 3D particle mesh representing Cospira's real-time connection network. Nodes = participants. Edges = live connections. The mesh breathes, pulses, and reacts to the mouse — never static, never repeating.
Geometry
1,200 nodes using THREE.Points with BufferGeometry. Edges rendered as THREE.LineSegments. Node positions stored in a Float32Array and updated each frame via Simplex noise displacement. Max edge distance: 120 units (proximity-based connection).
Mouse Parallax Response
Camera positioned at Z=400. On mousemove: camera.x lerps toward mouseNDC.x * 30, camera.y toward mouseNDC.y * -20. Lerp factor: 0.04 per frame. Result: 3D perspective shift without camera rotation — the mesh appears to float in front of the content.
Mobile Behaviour
On touch devices, replace mouse parallax with DeviceOrientationEvent beta/gamma. Clamp: ±15°. If permission denied or unavailable: ambient oscillation at 0.0008 rad/frame using a sine wave on both axes.
Scroll Behaviour
As user scrolls past hero section, mesh.position.z advances from 0 → -300 (depth recession) and opacity fades 1 → 0. Triggered via ScrollTrigger.create with scrub: 1.5. Creates a "diving into" the platform sensation.
⬡
7-Layer Parallax Architecture
▾
L1
Static noise grain (CSS pseudo)
Fixed SVG filter: feTurbulence baseFrequency="0.65" + feColorMatrix for monochrome grain. mix-blend-mode: overlay. Opacity 0.025. Animates at 8fps via CSS @keyframes offset.
L2
Deep background glow (0.05x scroll)
Two radial gradients: indigo-900/12 at top-left, violet-900/10 at bottom-right. Translate at 5% of scroll delta. Near-static — creates the sense of a light source infinitely far away.
L3
Three.js mesh canvas (0.15x scroll)
Canvas positioned absolute, pointer-events: none. Translates at 15% of scroll delta on top of its own internal camera drift. Creates double-parallax on the primary hero asset.
L4
Geometric accents (0.3x scroll)
6–8 CSS-drawn geometric shapes (rings, dots, partial arcs) positioned absolutely around the hero. Translate at 30% delta. Stagger their entrance reveals with 80ms intervals.
L5
Subheadline + live ticker (0.55x scroll)
Text content block. Moves at 55% of scroll delta. Fades out by scroll position 0.3× viewport height.
L6
Primary headline (0.8x scroll)
Display type. Moves slowest of all text layers — creates the illusion that the headline is physically closer to the viewer than the subtext.
L7
CTA cluster (1.0x scroll — anchored)
Standard scroll speed. CTAs appear to be "resting" on the page surface while everything around them floats at different depths.
Module 03 · Interaction & UX Micromoments
The Tactile Digital Layer
Every hover, click, and cursor movement must feel like touching a physical object
⊹
Custom Cursor System — 4 Morphing States
▾
①
Default State — Dual Ring
Outer ring (28px): 1px indigo border, follows at 80ms lag (lerp 0.12). Inner dot (6px): tracks true cursor at 0ms lag. Gap between them visualises cursor velocity — rings compress at speed.
②
CTA Hover — Text Morph
On hover over any .cta element: outer ring scales to 64px, fills to rgba(255,255,255,0.08). Inner dot fades. A context word appears inside the ring: ENTER on primary CTAs, PLAY on game cards, WATCH on video features. Text morphs using clipPath wipe.
③
Link Hover — Inversion
On hover over text links: cursor inverts. Outer ring fills solid white. Inner dot becomes dark. Uses mix-blend-mode: difference on the cursor element — automatically inverts whatever colour is beneath it without any JS colour detection.
④
Drag / Scroll Zones — Arrow Morph
Over the horizontal game arcade strip: cursor morphs to a horizontal double-arrow (← →) at 36px. Over the Three.js canvas: cursor disappears and the inner dot becomes a 3px crosshair matching the mesh colour.
◉
Magnetic Hover System
▾
Implementation
Every .magnetic element (buttons, nav links, social icons) has a detection zone 1.5× its bounding box. On mousemove within zone: element translates toward cursor at 28% of cursor offset. Spring return on mouseleave using gsap.to with stiffness: 200, damping: 18.
function initMagnetic(el) { const strength = 0.28; el.addEventListener('mousemove', (e) => { const r = el.getBoundingClientRect(); const dx = e.clientX - (r.left + r.width / 2); const dy = e.clientY - (r.top + r.height / 2); gsap.to(el, { x: dx * strength, y: dy * strength, duration: 0.3, ease: 'power2.out' }); }); el.addEventListener('mouseleave', () => { gsap.to(el, { x: 0, y: 0, duration: 0.6, ease: 'elastic.out(1, 0.4)' }); }); }
⚡
Haptic-Like Click Feedback
▾
A
Ripple origin burst
On click: inject a div.ripple at cursor position inside the button. Scale 0→1.8 in 400ms, opacity 1→0 in 400ms. pointer-events: none. Colour: 30% opacity of button's accent colour.
B
Button compression + spring return
transform: scale(0.94) on :active. Released via gsap.to with ease: "elastic.out(1.2, 0.5)" — slight overshoot creates physical spring feel.
C
Navigator vibration (mobile)
navigator.vibrate([8]) on primary CTA click on mobile. 8ms — imperceptible as a notification, perceptible as confirmation. Wrap in try-catch; fail silently if not supported.

Module 06 · Performance & Accessibility Contracts
60fps Is Not a Goal. It Is a Contract.
Every animation technique must pass the jank test before shipping
✓
GPU Acceleration Rules
▾
Composited properties only
All animations use only transform and opacity. Never animate width, height, top, left, margin, padding, or any layout property. Violating this causes layout recalculation and frame drops on every browser.
will-change discipline
Apply will-change: transform, opacity only to elements actively animating. Remove after animation completes via onComplete callback. Blanket application causes excessive GPU memory consumption.
Three.js render budget
Max draw calls: 4. Use THREE.InstancedMesh for all nodes (1200 instances, 1 draw call). Edge lines: single merged BufferGeometry. Texture-free — all visuals from vertex shaders. Antialiasing: false on mobile (saves ~15ms/frame).
Scroll event throttling
Never read scrollY in a scroll listener. Use ScrollTrigger's internal tick which reads scroll position once per rAF frame. For custom scroll effects: store latest scroll value in a variable in the listener, read it only in rAF callback.
⚠
Accessibility & Motion Sensitivity
▾
// Wrap ALL animation initialisation in this check const prefersReduced = window.matchMedia( '(prefers-reduced-motion: reduce)' ).matches; if (!prefersReduced) { initThreeJS(); initLenis(); initScrollTrigger(); initCursor(); } else { // Instant reveals, no scroll animations, no cursor FX document.querySelectorAll('.reveal') .forEach(el => el.style.opacity = '1'); }
Mobile graceful degradation
On touch devices: disable magnetic hover, disable custom cursor, simplify parallax to 2 layers max, reduce Three.js particle count to 400, disable grain overlay. Detect via ('ontouchstart' in window).
