Landing on a ladder top that is also a snake head: → NO further interaction. → Player stays at that cell. ``` ### Win condition ``` newPosition === 100 → phase = 'COMPLETE' winnerId = currentPlayerId emit GAME_WIN ``` ### Turn order ``` Players rotate in joinTimestamp ascending order. Spectators are skipped. Disconnected players are skipped (after 60s grace period). Default snake/ladder map (standard board) typescriptconst DEFAULT_SNAKES: Record = { 17: 7, 27: 1, 33: 18, 54: 34, 62: 19, 64: 60, 93: 68, 95: 24, 99: 78 }; const DEFAULT_LADDERS: Record = { 4: 14, 9: 31, 20: 38, 28: 84, 40: 59, 51: 67, 63: 81, 71: 91 }; ``` --- ## PHASE 5 — MOVEMENT RESOLUTION PIPELINE `MoveProcessor.resolve(state, playerId, diceValue)` executes atomically: ``` Step 1 Validate turn ownership → if state.currentPlayerId !== playerId: throw GameError('NOT_YOUR_TURN') Step 2 Validate diceValue → if diceValue < 1 || diceValue > 6: throw GameError('INVALID_DICE') Step 3 Check triple-six penalty → if consecutiveSixes === 2 && diceValue === 6: revert position, reset counter, advance turn emit TURN_CANCELLED Step 4 Entry check → if position === 0 && diceValue !== 6: advance turn, emit TURN_SKIPPED Step 5 Compute rawPosition = currentPosition + diceValue Step 6 Overshoot check → if rawPosition > 100: advance turn, emit OVERSHOOT Step 7 Win check → if rawPosition === 100: emit GAME_WIN, phase = COMPLETE, halt Step 8 Snake resolution (checked first) → if rawPosition in snakes: position = snakes[rawPosition], emit SNAKE_BITE Step 9 Ladder resolution (only if step 8 did NOT trigger) → if rawPosition in ladders: position = ladders[rawPosition], emit LADDER_CLIMB Step 10 Normal move → position = rawPosition, emit MOVE_COMPLETE Step 11 Extra-turn check → if diceValue === 6: increment consecutiveSixes, do NOT advance turn Step 12 Advance turn to next active connected player Step 13 Emit STATE_DELTA with full diff Every step is a pure function with typed input/output. Full resolution is reproducible from (GameState, diceValue) with zero side effects. Clients that attempt to emit `ROLL_RESULT` directly must be rejected at the server transport layer with `HOST_ONLY_ACTION`. --- ## PHASE 8 — HOST MIGRATION PROTOCOL ``` Trigger: host WebSocket disconnects (detected by server ping timeout, 5s) Step 1: Server marks hostId as disconnected in Redis Step 2: Server selects new host = min(joinTimestamp) among connected, non-spectator players Step 3: Server promotes new host → updates RedisRoomRegistry Step 4: New host derives new room key via HKDF (same masterSecret, new derivation nonce) Step 5: New host broadcasts HOST_MIGRATED event (signed with new key) Step 6: New host broadcasts full GameState snapshot (STATE_SYNC) Step 7: All clients verify new HMAC key, re-anchor sequenceId Step 8: Game resumes from current state — no moves lost If no eligible host exists (all players disconnected): room enters PAUSED state, preserved in Redis for 10 minutes. First reconnecting player becomes host and resumes.
